# OneSignal KPI Plan

## 集計の目的

OneSignal の登録、通知許諾、配信到達、クリックまでを同じ並びで見られるようにし、Push 通知の改善論点を日次で追える状態を作る。

## 2026-05-19 時点の初期値

| KPI | 値 | 補足 |
| --- | ---: | --- |
| 有効DL累計 | 33,855 | フォールバック値 |
| OneSignal登録数 | 29,408 | 登録率 86.8% |
| 通知許諾数 | 6,100 | DL基準許諾率 18.0% |
| OS基準許諾率 | 20.7% | 6,100 / 29,408 |
| 解除・無効化数 | 733 | 解除率 12.0% |
| 配信対象数 | 5,367 | 6,100 - 733 |
| 到達数 | 4,634 | 到達率 86.3% |
| クリック数 | 432 | CTR 9.32% |

## 優先して見るKPI

1. 通知許諾率
2. 配信対象数
3. 到達率
4. CTR
5. Push経由DL数

## まだ未確定のもの

- Push 経由 DL 数
- 有効DL累計の BigQuery ソース統合
- AppsFlyer や AppDB と突合した最終CV導線

## 進め方

1. スプレッドシートに日次スナップショットを蓄積する
2. HTML ダッシュボードで月次の見え方を揃える
3. GitHub Actions でデータ更新を自動化する
