1. ご指摘への直接回答

Q1. 「全件交渉に持ってきたが、全体の成約率は変わっていないのでは？」

全件で見ると確かに CR は B2 で低下（v3定義で A:42.8% → B2:40.0%, -2.8pt）。しかし §8で LP58（カババ） を除外して見ると、B2 のCRは 42.7% と A期 42.8% にほぼ並ぶ。 LP58 は 2025/12 から急増した新トラフィックで CR が構造的に低い（~20%）。これが混ざって全体CRを引き下げていた。

純粋な「絞り込み→全量持込」戦略の効果としては、A → B 延長（LP58除外）で CR 42.8% → 43.8%（+1.0pt 改善）。指摘者の感覚「変わっていない」は正しく、むしろ微増していた。

→ LP58除外で見ると CR はほぼ変わっていない／微増。「変わっていない」という指摘は妥当だった。

Q2. 「だから工数の無駄ではないか？」

Q1 の通り、CR は LP58除外で実は維持／微増。売上は +16.1% 純増（+366万/月）、出品も +10.6% 増、商談数も +40% 増えており、全体として明確な拡張に成功している。

確かに交渉成約率/商談数は 87.3% → 74% に下がっており、商談1件あたり売上も 15.85万 → 7.9万（限界）に低下。「同じ商談1件あたりの効率」は下がったのは事実。 しかし、CR は維持しつつ件数を増やして売上を +16% 伸ばせたなら、それは「工数の無駄」ではなく「productive な事業拡張」。

→ 工数の無駄ではない。LP58除外で CR 維持・売上 +16% 増を達成しており、戦略は明確に成功している。「商談1件あたりの効率は下がった」は事実だが、件数効果がそれを上回る。

Q3. では「全量持込」をやめるべきか？

続けるべき。LP58除外ベースで戦略は CR 維持・売上 +16% を達成している。やめれば月366万円・年4,400万円の売上を失う。

論点は「やめるか」ではなく、LP58（カババ）を本体KPI評価から切り離して別管理にすること。LP58はCRが構造的に低い（~20%）が、月187万円の追加売上を生んでいるので個別に採算判断する。

→ 戦略は維持。LP58を別建てトラフィックとして管理ラインを分けるのが次のステップ。

7. v3定義による完全再現と最新月延長 — 工数論点はこれが本筋

v3の「オク成約」定義（`highest_bid >= sold_out_price`、加算金を含まない）で再集計したところ、 v3レポートの全21ヶ月分の数値と完全一致。これを使って B 期を 2026/04 まで（10ヶ月） に延長し、§4までの申込月ベースの分析より 工数（=実際に商談リストに乗った案件）に直結する観点 で評価する。

v3定義の意味: 入札だけで売切を超えたケース = 純粋なオク成約（交渉担当の作業を要しなかった案件）。 加算金や売切引き下げで成約したケースは「交渉成約」に含める。本レポートの「工数」論点ではこの定義が正しい。 DealAnalysis.js / BQ_negotiation.js の canonical 定義（入札+加算金+自動車税 ≥ 売切）は v3 と数値が一致しないため、ここでは v3 定義を採用する。

8-4. LP58 の扱いに関する提言

  • LP58 は「別管理トラフィック」として扱う: 期待値が高く CR 構造的に低いため、本体 CR の評価対象に含めると見かけ上の数値が歪む
  • LP58 単独でも価値あり: 月187万円の追加売上（B2 5ヶ月平均）。CR 19% でも採算が合うなら継続価値はある
  • LP58 専用の交渉戦術／売切設定: CtoC 紹介で売主期待が高い前提を踏まえ、本体と異なる対応指針が必要
  • 本体KPI（CR・売上効率）の評価は LP58除外ベース に切り替えるべき

9. 結論と提案（LP58影響を踏まえた修正版）

戦略は成功。CRは LP58 除外で維持、売上は +16% 純増。論点は「やめるか」ではなく「LP58の別管理」。

分析を重ねた結果、初期に出した「工数の無駄」「B2でCR急落」という強い結論は LP58（カババ）混入によるアーティファクト だった。 LP58 を除いた純粋な戦略効果としては CR 42.8% → 43.8%（+1.0pt 改善） ・ 売上 +16.1%（月+366万、年+4,400万）。 指摘者の「成約率は変わっていない」は正しく、ただし「だから工数の無駄」は誤り。CR を維持しつつ件数を伸ばし、売上を増やせたのが実態。

提案:

  • 即日: 本体KPIをLP58除外ベースで定義し直す。月次レポートのCR・売上効率指標は LP58 を分けて表示。混ぜると見かけ上の劣化が出て誤判断につながる。
  • 即日: LP58 専用の採算ライン を設定。CR 19%・売上187万/月で採算合うか、別ルールでの売切設定や交渉手法が必要か検討。
  • 短期: B1（CR 46.5%）→ B2 LP58除外（42.7%）の -3.8pt 差を分解（季節か、難案件混入か、市況劣化か）。B1が異常値だった可能性も含めて検証。
  • 短期: 全量持込戦略の継続を意思決定者に明示。「やめれば月366万・年4,400万の売上を失う」。
  • 中期: LP58 と同様の「期待値高め系」流入経路が今後も増える可能性に備え、トラフィック別のCR管理体制を整備。
  • 中長期: 出品1件あたり売上をB1水準（6.20万）に戻す施策（売切適正化、申込時トリアージ）は LP58除外の本体ラインで進める。
分析の経緯（透明性のため記録）:
  • v3レポートのフォローアップとして開始 → 当初「申込月ベースで限界CR 1.2% = 工数の無駄」と結論
  • v3定義（出品月ベース・商談月ベース）で再集計 → 限界CR 41% に修正、「無駄ではないが効率は下がっている」へ
  • LP58除外で再々集計 → 「CR維持・売上+16%」が判明、「工数の無駄」結論は撤回
  • 教訓: 分析ベース（申込/出品/商談月）と母集団（全件/LP58除外）を切り替えるだけで結論が大きく変わる。最終判断は必ず複数の切り口で検証すること。
本レポートで答えていない論点:
  • 商談数（negotiation_log）の LP58 除外: `negotiation_log.deal_number`（"A..."形式）と `all_deals_view.application_number`（"G..."形式）が直接 JOIN できない。商談数は LP58 込みのまま。
  • LP58の採算分析（粗利ベース）: 売上187万/月の内訳と LP58 専属対応に要する工数を比較する必要あり。
  • 担当者別の工数推移（B1→B2でLP58案件を誰が捌いていたか）。担当者分析と組み合わせる必要あり。
  • 「ローン残債のみNG」というB期フィルタの実装が all_deals_view 側で識別できないため、フィルタ運用の実態は申込→出品の歩留まりから推定している。
