# 2026-04-08 Research: ストアPV→DL率の改善（ASO）

## 1. 結論（先に）
- iOSは **AI査定を最初の静止アセットで即理解させる** 方針が妥当。Appleはカスタムプロダクトページでスクリーンショット・プレビュー・プロモーションテキストを変えられ、参照流入では平均 +2.5pt のCVR改善を公表している。  
  出典: https://developer.apple.com/app-store/custom-product-pages/ （閲覧日: 2026-04-08）
- Androidは **出費管理 + 将来予測** を独立訴求できるカスタムストア掲載情報を使うべき。Google Playは国・URL・Google Ads・検索キーワードなどでカスタム掲載情報を出し分けられ、最大50ページまで作成できる。  
  出典: https://support.google.com/googleplay/android-developer/answer/9867158?hl=en （閲覧日: 2026-04-08）
- 計測は **ストア訪問→獲得のCVRをプラットフォーム標準の定義で追う** のがよい。Google PlayのConversion analysisは store listing visitors / acquisitions / conversion rate を直接見られ、同業比較のベンチマークも使える。  
  出典: https://support.google.com/googleplay/android-developer/answer/9859173?hl=en （閲覧日: 2026-04-08）
- iOSでアプリプレビューを使う場合、**プレビューがスクリーンショットより先に表示される**。つまり「1枚目スクショで伝える」設計は、プレビュー有無とセットで考える必要がある。  
  出典: https://developer.apple.com/help/app-store-connect/manage-app-information/upload-app-previews-and-screenshots （閲覧日: 2026-04-08）
- Androidでは **スクリーンショット自体が検索やホーム面にも露出しうる** ため、B案/C案も単体で意味が通る必要がある。Google Play公式はスクリーンショットが検索やホーム、ストア掲載情報で表示されることを案内している。  
  出典: https://support.google.com/googleplay/android-developer/answer/1078870?hl=en （閲覧日: 2026-04-08）

## 2. 根拠メモ（2026-04-08確認）

### 2.1 Apple App Store
- AppleのCustom Product Pagesは、スクリーンショット・プロモーションテキスト・アプリプレビューを変えた追加ページを最大70個まで作成できる。検索結果にも表示可能。  
  出典: https://developer.apple.com/app-store/custom-product-pages/ （閲覧日: 2026-04-08）
- Appleは「参照流入をカスタムプロダクトページに送ると、平均で +2.5 percentage points、デフォルト商品ページ平均 1.6% に対して 156% 増」と明記している。  
  出典: https://developer.apple.com/app-store/custom-product-pages/ （閲覧日: 2026-04-08）
- Product Page Optimization では、元ページに対して **最大3つの別バージョン** をテストできる。テスト対象は app icon / screenshots / app previews。  
  出典: https://developer.apple.com/app-store/product-page-optimization/ （閲覧日: 2026-04-08）
- カスタムプロダクトページのApp Analytics指標は、**そのページが first-time downloads を5件以上獲得すると使える**。  
  出典: https://developer.apple.com/help/app-store-connect/create-custom-product-pages/configure-multiple-product-page-versions/ （閲覧日: 2026-04-08）
- App Store Connectでは、iPhone / iPad / Mac / Apple TV で **app previews always precede screenshots** と案内されている。  
  出典: https://developer.apple.com/help/app-store-connect/manage-app-information/upload-app-previews-and-screenshots （閲覧日: 2026-04-08）

### 2.2 Google Play
- Google PlayのCustom store listingsは、**specific user segments** 向けに出し分けできる。対象条件は国、unique URL、Google Ads、検索キーワード、inactive users など。  
  出典: https://support.google.com/googleplay/android-developer/answer/9867158?hl=en （閲覧日: 2026-04-08）
- Custom store listingsは **最大50ページ**。各ページで app name / icon / descriptions / graphic assets を個別最適化できる。  
  出典: https://support.google.com/googleplay/android-developer/answer/9867158?hl=en （閲覧日: 2026-04-08）
- Google Play公式は、スクリーンショットについて **best features をハイライトし、テキストは必要最小限、テキスト入り素材は言語ごとに別で用意** することを推奨している。  
  出典: https://support.google.com/googleplay/android-developer/answer/13393723?hl=en （閲覧日: 2026-04-08）
- Google Play公式は、スクリーンショットが **search / homepage / store listing** で表示される可能性を明示している。また preview video があると、その後ろに screenshots が並ぶ。  
  出典: https://support.google.com/googleplay/android-developer/answer/1078870?hl=en （閲覧日: 2026-04-08）
- Conversion analysis では、store listing visitors / acquisitions / conversion rate を traffic source, store listing, country, language, UTM などで切り分けられる。さらに **conversion rate benchmarks** で peer 比較も可能。  
  出典: https://support.google.com/googleplay/android-developer/answer/9859173?hl=en （閲覧日: 2026-04-08）

## 3. UcarPACへの適用方針
- A案（iOS向け）は **AI査定 + 個人情報なし + 将来予測** を1枚目に集約する。Apple側でプレビューを置くなら、プレビューのポスターフレームまで同じメッセージに揃える。
- B案（Android向け）は **出費管理 + 将来予測** を、家計管理アプリに近い理解負荷で見せる。Google Playでは intent 別に custom store listing を切り、広告や検索キーワードごとに訴求を変える余地が大きい。
- C案（競合比較）は **「相場チェックなら電話0本」** と限定して使う。売却相談フェーズまで含めた包括表現にすると、今回の社内コンテキスト上の「電話1社」とずれるため、UI上で `相場を見る / 売る相談に進む` を切り替える表現が安全。
- 推定: 現状17.3%から25%へ引き上げるには、1枚目で「何ができるか」、2枚目で「なぜ残す価値があるか」、3枚目で「なぜ他サービスより安心か」を順番に潰す構成が妥当。これは上記のApple/Google両ストアで、スクリーンショット差し替えと個別計測が可能であることから導いた設計仮説。

## 4. 今回のプロトタイプ反映
- スクショA: iOS向け。AI査定を主役にし、価格・将来予測・個人情報なしを同居させる。
- スクショB: Android向け。出費管理と将来予測を1画面で見せ、今すぐ売らない層のDL理由を作る。
- スクショC: 競合比較。「電話0本」は相場チェック時だけに限定し、売却相談時は1社対応へ切り替わるUIにする。
- テキスト量は最小限にし、数字と短いタグラインで理解させる。Google Play公式の「keep text to a minimum」に合わせる。
