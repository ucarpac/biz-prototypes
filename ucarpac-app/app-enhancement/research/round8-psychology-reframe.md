# Round 8: 「下がる価格を見せる」問題の根本解決

## 核心的な問題

中古車は減価償却資産。価格は基本的に下がり続ける。
株式・不動産のように「上がるかもしれない」期待がない。
→ **価格トラッキングは本質的にネガティブな体験**になる。

### 実証データ
- 自己トラッキング研究: ネガティブ指標を見続けると**活動の楽しさが減少**し、**1/3が6ヶ月以内に放棄** [Level B]
- Carvanaユーザー: 「1ヶ月で$6,000下がった」「25%変動で使い物にならない」と不満 [Level B]
- 損失回避バイアス: 損失の痛みは利得の喜びの**約2倍** [Level A: Kahneman & Tversky]

## フレーミング転換: 3つの代替アプローチ

### アプローチ1: 「価格」→「手残り額（エクイティ）」に変換

**概念**: 車両価値 - ローン残債 = 手残り額（エクイティ）

| 状態 | 表示例 | 心理効果 |
|------|--------|---------|
| ポジティブエクイティ | 「今売ると105万円手元に残ります」 | 具体的な「もらえる額」=ポジティブ |
| ゼロエクイティ | 「ローン完済ちょうどの金額です」 | ニュートラル |
| ネガティブエクイティ | 「ローン残債が車両価値を25万円上回っています」 | 危険信号→行動喚起 |

**ローンなしの場合**: 手残り = 売却額そのまま。「今売ると285万円が手元に入ります」

**米国での実装例**:
- Bankrate, Chase等が「エクイティ計算機」を提供 [Level A]
- 「positive equity」=売却のチャンス、「negative equity」=要注意、と明確に分類
- UcarPACの強み: ローンシミュレーション機能で残債データを既に保有

### アプローチ2: 「いくら下がったか」→「今が最高値」に変換

**概念**: 中古車は時間とともに下がるので、**常に「今日が最も高い日」**

| NG（現行の発想） | OK（転換後） |
|--------------|-----------|
| 「先月より3万円下がりました」 | 「今日が一番高い日。来月にはさらに下がる見込みです」 |
| 「残価率57%に低下」 | 「あと1年で残価率が50%を切る見込み。今なら57%」 |
| 「年間維持費68万円」 | 「これから1年持つと68万円かかります。今売れば285万円」 |

**行動経済学の裏付け**:
- フレーミング効果: 同じ情報でも提示の仕方で判断が変わる [Level A]
- 損失回避: 「今売らないと〇万円損する」は「今売ると〇万円得する」より2倍効果的 [Level A]
- **ただし**: 損失フレームの多用はアプリへのネガティブ感情につながるリスク
- **バランス**: 「今が最高値」はポジティブフレーム+損失回避の組み合わせ

### アプローチ3: 「価値の下落」→「次の車への準備」に変換

**概念**: 売却はゴールではなく、**次の車へのステップ**。価格下落ではなく「乗り換え計画」として位置づける。

| 要素 | 表示例 |
|------|--------|
| 乗り換えシミュレーション | 「今のBMWを売却 → 頭金285万円で次の車へ」 |
| 月々の支払い変化 | 「乗り換えると月の支払いが2.3万円→1.8万円に」 |
| 同車種オーナーの動向 | 「BMW 3シリーズオーナーの35%が3年目で乗り換え」 |
| 次の車おすすめ | 「売却額285万円で狙える人気車種」 |

**マーケティング観点のメリット**:
- 「次の車」への関心 = アプリを開く前向きな理由
- UcarPACプラットフォームで次の車も買う可能性 → KGI直結
- ディーラーへの送客チャンスも生まれる

## UcarPACの固有データで実現できること

### 既存データ（コードベースから確認済み）
1. **車両情報**: メーカー・モデル・グレード・年式・色・走行距離
2. **ローン残債**: RefinanceSimulationから取得可能
3. **査定額**: AI査定結果
4. **維持費データ**: CostSimulationの6カテゴリ
5. **オークション実績**: UcarPAC独自のデータ（他社にない）

### これらから生成できる「前向きな情報」
| データ | 変換 | 表示 |
|--------|------|------|
| 査定額 - ローン残債 | 手残り額 | 「今売ると105万円が手元に」 |
| オークション入札数 | 需要度 | 「あなたの車種に12社が注目中」 |
| 同車種の売却実績 | 社会的証明 | 「同じ車の35%が3年目で売却」 |
| 査定額 + 維持費 | 乗り換えプラン | 「売却額を頭金に月1.8万円で乗り換え可能」 |
| 季節変動データ | タイミング | 「3月は需要期。今が年間で最も高い時期」 |
| リセールバリューランキング | 誇り/危機感 | 「BMWの残価率は輸入車で3位」 |

## 設計原則の再定義

### NG: やってはいけないこと
1. ❌ 「下がりました」通知（ネガティブ体験の押し付け）
2. ❌ 価格推移の下降グラフを大きく表示（視覚的にネガティブ）
3. ❌ 損益分岐点グラフ（概念が不明確+ネガティブ）
4. ❌ 残価率の数字を大きく表示（下がり続ける指標）

### OK: やるべきこと
1. ✅ 手残り額（エクイティ）を主指標に
2. ✅ 「今日が最高値」フレーミング
3. ✅ 「〇社が注目中」（需要の可視化=ポジティブ）
4. ✅ 乗り換えシミュレーション（前向きなゴール）
5. ✅ 同車種オーナーの動向（社会的証明）
6. ✅ 維持費の節約提案（アクション可能なポジティブ情報）
7. ✅ リセールバリューランキング（相対的な位置づけ）

## 「クルマの家計簿」との根本的な違い（再定義）

| | クルマの家計簿 | UcarPAC v4 |
|---|---|---|
| 主指標 | 維持費の総額（蓄積型） | **手残り額**（行動に直結） |
| 心理 | 「こんなにかかってる」（ネガティブ気味） | 「今売ると〇万円手元に」（ポジティブ） |
| 行動喚起 | なし（記録が目的） | **売却 or 乗り換え or 節約** |
| 差別化 | コスト記録の網羅性 | **リアルタイム査定額 × ローン残債 × 維持費 → 意思決定** |
