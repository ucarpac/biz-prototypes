# 最悪シナリオ（Worst Case）

> 調査結論に基づいて全機能を実装した場合に想定される最悪の帰結を描く。

---

## シナリオ概要

UcarPACが調査結論に従い、以下の全てを実装する:
1. 車ライフサイクル管理へのリポジショニング（UI/UX全面改修）
2. メンテナンスリマインダー・車検通知
3. 燃費・維持費管理機能
4. 価格アラート強化 + 市場トレンド通知
5. オンボーディングの再設計
6. ゲーミフィケーション要素

---

## 最悪シナリオ: 「全部入りアプリの自滅」

### Phase 1: 開発の泥沼化（0-6ヶ月目）

**発生条件**: 経営層が「全部やろう」と判断し、開発チームに全機能の同時実装を指示。

- セルフ査定機能の開発と並行して上記機能の開発が始まる
- スコープクリープが発生し、当初3ヶ月の予定が6ヶ月に延長
- 開発コストが想定の1.5-2倍に膨張
- 既存のバグ修正・セキュリティアップデートが後回しにされる
- 開発チームのモチベーション低下、離職リスクの増大

### Phase 2: リリース後の混乱（6-9ヶ月目）

**発生条件**: 機能をまとめてリリースしたが、ユーザーテストが不十分。

- 既存ユーザーがアップデート後のUIの変化に戸惑う
- 「査定アプリなのにメンテナンス記録？意味がわからない」というレビューが増加
- App Storeの評価が4.0から3.2に低下
- 新機能のバグが多発し、クラッシュ率が上昇
- メンテナンスリマインダーを登録するユーザーは全体の5%未満に留まる
  - 理由: 手動入力が面倒、既にオートバックスアプリを使っている

### Phase 3: 逆効果の顕在化（9-12ヶ月目）

**発生条件**: ユーザーへのプッシュ通知を月2回に増加。

- 「車の価格が下がりました」通知がネガティブ体験を生む
- 「メンテナンス時期です」通知は未登録ユーザーには届かない
- 通知の不要感からアプリの通知を全てOFF、またはアンインストール
- **削除率が60%→70%に悪化**
- 新規インストール数も減少（ストア評価低下の影響）

### Phase 4: 戦略の見直しと損失確定（12-18ヶ月目）

**発生条件**: KPIの悪化が明確になり、経営判断が求められる。

- 追加機能の利用率が5%以下であることが判明
- 「ライフサイクル管理」コンセプトの撤回を検討
- しかし、既にUI/UXを全面改修しているため元に戻すにも工数が必要
- セルフ査定機能のリリースが6ヶ月遅延しており、競合に先行されるリスク
- 開発チームの中核メンバーが1-2名離職

### 最終的な損害

| 項目 | 想定損害 |
|------|---------|
| 開発コスト超過 | 当初予算の50-100%超過 |
| セルフ査定機能の遅延 | 6ヶ月以上 |
| アプリ削除率 | 60% → 70%に悪化 |
| App Store評価 | 4.0 → 3.2 |
| 新規インストール数 | 20-30%減少 |
| 開発チーム離職 | 1-2名 |
| ブランドイメージ | 「車査定」の専門性が希薄化 |

## 最悪シナリオの発生確率評価

| 要素 | 発生確率 | 根拠 |
|------|---------|------|
| スコープクリープによるコスト超過 | 高（60%以上） | 業界統計で大規模機能追加の過半数がスコープクリープを経験 |
| 追加機能の低利用率 | 高（70%以上） | ソフトウェア機能の80%は「ほとんど使われない」（Pendo調査）|
| 通知による逆効果 | 中（40%程度） | 通知内容の設計次第で回避可能だが、車の価格下落通知はリスク |
| App Store評価低下 | 中（30%程度） | 段階的リリースで軽減可能 |
| 開発メンバー離職 | 低-中（20-30%） | プロジェクト管理の質に依存 |

**総合評価**: 上記シナリオの全てが同時に発生する確率は低い（10%程度）が、**部分的な失敗（特にスコープクリープと低利用率）の発生確率は非常に高い（70%以上）**。

---

## 回避のために検討すべきこと

最悪シナリオを回避するための条件（反論書の立場を維持しつつ）:

1. **機能追加の前に、削除理由の一次データを取得する**。推測で開発投資を行わない。
2. **「全部やる」ではなく、1機能ずつの仮説検証**。MVP（最小実行可能製品）で効果測定してから次に進む。
3. **LINEミニアプリ等の代替チャネルを先に検証する**。ネイティブアプリの機能拡充が最適解とは限らない。
4. **セルフ査定機能を最優先にする**。リテンション施策はセルフ査定リリース後のデータを見てから判断しても遅くない。
5. **「定着率」ではなく「売却転換率」をKPIに設定する**。定着率は手段指標であり、目的指標ではない。
