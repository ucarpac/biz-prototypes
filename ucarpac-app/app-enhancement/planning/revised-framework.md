# 修正フレームワーク: KGI起点の車両ライフサイクルプラットフォーム戦略

## ゴール構造の再設定

```
KGI: プラットフォームでの売買取引量の最大化
 ├── KPI-1: アプリ削除率の改善（60%→目標40%以下）
 ├── KPI-2: セルフ査定への転換率向上
 ├── KPI-3: 月次アクティブユーザー率（MAU/Install）向上
 └── KPI-4: 周辺サービス送客による収益化（新規収益源）

手段:
 └── 車両ライフサイクル管理プラットフォーム化
     ├── 保有期間中の継続接点（= 削除防止）
     ├── 売却判断の材料提供（= セルフ査定への導線）
     └── 車両付随データ活用（= マーケティング・送客収益）
```

## 前回のプロトタイプに欠けていた視点

1. **マーケティングファネルの設計**
   - 車両データ → マーケティングシグナル → パーソナライズ提案 → 送客/CV

2. **収益チャネルの具体化**
   - 保険見積もり送客（最大の収益機会）
   - 車検予約送客
   - メンテナンス予約送客
   - リファイナンス提案

3. **既存機能の活用**
   - コストシミュレーション → データは入力させてるが「提案」に変換できてない
   - ローン/リファイナンス → 借り換え先への送客
   - V-Point → ロイヤルティ強化のレバー

4. **データ駆動の「賢い提案」**
   - 「あなたの保険料は相場より高いかもしれません」
   - 「車検費用15万円 vs 今売れば285万円」
   - 「月間走行500km以下なら走行距離連動保険がお得」

## 新プロトタイプの設計方針

### コンセプト: 「車の家計簿 + 賢いアドバイザー」

ユーザーが入力したコストデータを**受動的に蓄積するだけでなく、能動的にアドバイスに変換**する。

### 画面構成

1. **ダッシュボード（改良版）**
   - 車両価値 + 維持費の対比ビジュアル
   - 「今月のインサイト」カード（AI提案）
   - セルフ査定CTA

2. **コストトラッカー（既存機能の強化）**
   - タイムライン形式のコスト記録
   - カテゴリ別コスト分析（保険/車検/燃料/駐車場/ローン）
   - 「節約チャンス」ハイライト → 送客

3. **スマート通知 & インサイト**
   - データ駆動の提案通知
   - 保険見直し提案
   - 車検 vs 売却コスト比較
   - メンテナンスリマインダー

4. **サービスマーケットプレイス**
   - 保険見積もり（楽天/インズウェブ等へ送客）
   - 車検予約（楽天Car車検等へ送客）
   - ロードサービス
   - リファイナンス（提携金融機関）

5. **マイカーカルテ**
   - 車両情報 + メンテナンス履歴
   - 価格推移グラフ
   - 所有コストサマリー
   - 「いつ売るべきか」AI予測
