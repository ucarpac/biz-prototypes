# Round 10: 価格提示後のファネル設計 — ウルトラシンク

## 現実のファネル（確認済み事実）

```
広告 → インストール → 車両登録 → 価格を見る
  ├── 3%: AI査定 or セルフ査定 → 売買相談申込（CV）
  └── 97%: ???
       ├── コストシミュレーションを使う（何%かは不明）
       ├── ローンシミュレーションを使う（何%かは不明）
       └── 何もせず離脱 → 60%が削除
```

## 核心的な問い

**「価格を見た後、売らないと決めたユーザーに、次に何をさせるか？」**

### 思考1: ユーザーの心理状態を分類する

価格を見た瞬間のユーザーは、以下のどれかの心理状態にいる：

| 心理状態 | 割合（推定） | 次の行動 |
|---------|------------|---------|
| A. 「思ったより高い！売ろう」 | 3% | → AI査定/セルフ査定/相談（現行CVR） |
| B. 「思ったより安い…売れない」 | 20-30%? | → ガッカリ → **ここで離脱する人が最多では？** |
| C. 「まあこんなもんか。今は売らない」 | 30-40%? | → 「今は売らないけど将来的に…」 |
| D. 「相場を知りたかっただけ」 | 20-30%? | → 目的達成 → アプリ不要 → 削除 |

### 思考2: 各心理状態に対して何を出すべきか

#### A. 「売ろう」（3%）→ 現行通り。触らない。
- AI査定 → セルフ査定 → プラットフォーム出品
- ここは完成している

#### B. 「安い…売れない」（最大の離脱ポイント）
- **この層が60%削除の主因では？**
- 「安い」と感じる理由:
  - 購入価格との差が大きい（損失回避バイアス）
  - ローン残債より低い（ネガティブエクイティ）
  - 期待値とのギャップ
- **この層に出すべきもの**:
  - 「年間コスト91万円。同クラス平均105万円より14万円お得」→ **「安いけど、維持は賢くできてる」というポジティブ要素**
  - 「ローン残債180万円。あと18ヶ月で売却可能ゾーンに」→ **売り時の見通し**
  - 「3月の決算期には価格が上がる傾向あり」→ **希望のある情報**
  - **結論: 「今は売れないけど、いつか売れるタイミングが来る。それまでこのアプリで準備しよう」**

#### C. 「今は売らない」（最大のポテンシャル層）
- **この層をどう定着させるかが、追加機能のターゲット**
- 売らない理由は様々：
  - まだ乗りたい
  - 次の車が決まってない
  - タイミングが悪い
- **この層に出すべきもの**:
  - コストシミュレーション → 「持ち続けるコストを把握しよう」
  - 年間コスト比較 → 「あなたの車は維持費が安い方？高い方？」
  - メンテナンスリマインダー → 「車の価値を維持するために」
  - 定期的な価格チェック → 「売り時が来たらお知らせ」

#### D. 「相場を知りたかっただけ」
- **この層の削除は防げない。防ぐべきでもない。**
- ただし、1つだけできること:
  - 「車検前に価格が変わります。通知を受け取りますか？」
  - メールorLINEで繋がりを維持（アプリ削除しても接点が残る）

### 思考3: 「価格提示後の分岐UI」の設計

現行は価格を見た後、ユーザーが自分でAI査定/コストシミュレーション等を探す必要がある。
**「次に何をすべきか」をアプリ側から提案すべき。**

```
価格表示: 285万円
     ↓
┌──────────────────────────────────────┐
│  この価格をどう活かしますか？           │
│                                      │
│  [1] 売却を検討する                    │
│      → セルフ査定で最高値を目指す       │
│      → 最大8,000社が入札              │
│                                      │
│  [2] もっと詳しい査定を受ける           │
│      → AI査定で精度を上げる            │
│      → 写真を送るだけ                 │
│                                      │
│  [3] 維持費を確認する                  │
│      → 年間いくらで乗れるか計算        │
│      → 同クラスとの比較もできます       │
│                                      │
│  [4] 価格変動を追跡する                │
│      → 売り時が来たら通知します        │
│      → 車検前のおすすめタイミングも     │
└──────────────────────────────────────┘
```

### 思考4: 選択肢の設計原則

1. **「売る」を最上位に**（KGI直結。3%のCVRを守る）
2. **「詳しく知る」を2番目に**（AI査定→コストシミュレーション。エンゲージメント深化）
3. **「維持費を確認」を3番目に**（コストシミュレーションへの導線。データ入力の動機）
4. **「追跡する」を4番目に**（通知許可の取得。長期接点の確保）

### 思考5: 選択肢ごとの裏側の設計

#### [1] 売却を検討する → 現行フロー
- セルフ査定 → プラットフォーム出品 → 成約
- **変更なし**

#### [2] AI査定 → コストシミュレーションへの導線
- AI査定結果表示後に「この車の年間維持費も計算できます」
- AI査定 → コストシミュレーション → 「年間91万円。同クラスより14万円お得」
- **AI査定のデータがコストシミュレーションに自動連携**

#### [3] 維持費を確認する → 成績表への導線
- コストシミュレーション入力 → 年間コスト算出
- 同クラス比較（ザイム型） → 「保険料が平均より高い→見直しチャンス」
- **コスト入力 = データ蓄積 = マーケティングシグナル**

#### [4] 価格変動を追跡する → 通知設定
- プッシュ通知許可を取得
- 車検日入力（任意）
- メールアドレス/LINE連携（アプリ削除後も接点維持）
- **月次レポートで再訪問を促す**

### 思考6: マーケティング観点

各選択肢から得られるデータ:

| 選択肢 | 取得データ | マーケティング活用 |
|--------|----------|-----------------|
| [1] 売却検討 | 売却意思あり | 即セルフ査定→CV |
| [2] AI査定 | 車両詳細+写真 | 精密なリード情報 |
| [3] 維持費確認 | 保険料/ローン/駐車場 | 保険見直し送客、リファイナンス提案 |
| [4] 追跡する | 通知許可、メール | 長期CRM。車検前に再アプローチ |

**全選択肢がマーケティングデータの入口になっている。**
ユーザーはどれを選んでも自分にとって価値がある行動をしているが、UcarPAC側にもデータが蓄積される。

### 思考7: 離脱をどう防ぐか

#### 「何も選ばない」ユーザーへの最後の一手
- 「価格変動があったらお知らせしましょうか？」のポップアップ
- メールアドレスだけでも取得 → アプリ削除されても接点維持
- **LINE友だち追加**で削除問題をバイパス

#### コストシミュレーション後の離脱防止
- 「年間コスト91万円」を見た後の分岐:
  - 「同クラスと比較する」→ 成績表（ザイム型）
  - 「維持費を下げる方法を見る」→ 保険見直し等の提案
  - 「いつ売るべきか確認する」→ ローン×価格グラフ → 売り時判定

## 結論: ファネル設計の要約

```
価格表示（285万円）
     ↓
「この価格をどう活かしますか？」（4択）
     ↓
[1] 売却 → セルフ査定 → CV（3%）
[2] AI査定 → 詳細データ取得 → コスト計算へ誘導
[3] 維持費確認 → コスト入力 → 成績表 → 節約提案
[4] 追跡 → 通知許可 → 月次レポート → 将来のCV
     ↓
各フローの出口で:
「ところで、LINE友だち追加しませんか？」
     ↓
アプリ削除されても接点維持
```
