# Round 11: 全ラウンド統合 — 最終設計

## これまでの気づきの整理（残すべきもの）

### 確定した設計原則（全ラウンドの結論）

1. **価格表示はメイン。即決型3%の導線を妨げない**（Round 10, challenger）
2. **追加機能は「売らなかった97%」のフォールバック**（Round 10）
3. **年間コスト（価値下落+維持費）は既存実装。これを強化**（Round 9, ユーザー指摘）
4. **ザイム型の同車格比較で「面白さ」を付加**（Round 9, ユーザー発案）
5. **ローン残債×買取相場の交差で売り時判定**（Round 9, 既存グラフの強化）
6. **広告臭排除: マーケットプレイスタブ不要。送客はコンテキスト埋め込み**（Round UX review）
7. **スケジュールはデフォルト推定値で入力最小化**（Round 7, challenger）
8. **リコール通知: 入力ゼロで価値提供**（Round 7）
9. **固定費自動記録: 保険・ローン・駐車場は一度入力で毎月自動**（Round 7）
10. **「下がる価格を毎回見せる」のネガティブ体験に注意。ただし価格自体は隠さない**（Round 8, challenger）
11. **意志確定トリガーは「次の車が決まること」。コスト比較ではない**（Round 10 challenger）
12. **LINE連携はアプリ削除問題のバイパスとして有効**（Round 6）
13. **V-Pointインセンティブでデータ入力率向上**（Round 6）
14. **プログレスバーで入力完了率+55%（LinkedIn事例）**（Round 6）

### 残った未解決課題

1. **買取相場データのカバレッジ不足** → レンジ表示 or AI査定CTAで対応
2. **31-90日熟考層がなぜ待つのか** → インタビューが必要（アプリ外要因の可能性）

## 乗り換えシミュレーションの詳細設計

### データソース（UcarPAC既存マスタ）
- car_familiesマスタ: メーカー・モデル・グレード
- 排気量データ: 税額算出
- 燃費データ: 年間燃料費算出
- リセールバリュー: 将来価値の推定
- 査定実績: 売却予想額

### 計算ロジック
```
【現在の車】
  売却予想額: 285万円
  年間コスト: 91万円（価値下落23万 + 維持費68万）

【候補の車（例: トヨタ RAV4）】
  購入価格: 350万円
  頭金（売却額）: 285万円
  追い金: 65万円
  年間コスト（推定）: 78万円（価値下落15万 + 維持費63万）

【比較結果】
  年間コスト差: -13万円/年 お得に
  燃費差: 12km/L → 16km/L（年間3.2万円節約）
  排気量: 2000cc → 2000cc（税額同じ）
  リセールバリュー: 輸入車3位 → 国産車5位
```

### UI設計
- 「今の車」と「次の車」を左右で比較
- 候補車はUcarPACのリセールバリューランキングからレコメンド
- 「年間○万円お得になります」のサマリー
- 「この車を見つけたらUcarPACに相談」CTA → プラットフォーム誘導
